你有多少个"待办"活在脑子里？

不是写在纸上的那种。是那种你反复想起又反复忘掉的事——那篇该写但没写的文章、那个该聊但没聊的人、那个说"回头研究一下"就再也没碰过的方向。

我有过 6 个。一天之内。

你知道那种感觉——明明没做什么，但就是累。这不是精力问题，是系统问题。

David Allen 在 2001 年说过：**你的大脑是用来产生想法的，不是用来存储想法的。** 二十多年后，这句话在 AI 时代变得更有攻击性了——因为现在你不仅有工具"记"，还有工具"管"。

如果你已经在用 AI 帮你写代码、写邮件、做研究，那你已经比 90% 的人多了一把瑞士军刀。但有一个问题你可能还没认真想过：**昨天的 AI 对话产出，今天还能直接复用吗？**

大多数人的答案是：不能。AI 聊完就没了。知识散落在 Claude 的会话记录、飞书消息、网页收藏夹里。下一次遇到类似问题，你重新描述上下文，重新让 AI 理解你的处境——每一轮对话都是从零开始。

这篇文章聊的，就是怎么把这个循环打断。

## GTD + OKF：一套系统，两个引擎

**GTD**（Getting Things Done）管的是执行闭环——把所有"未完事项"从大脑赶出去，交给一个你信得过的外部系统。大脑不再当硬盘，只当 CPU。

**OKF**（Open Knowledge Format）管的是知识沉淀——事情做完后留下的洞察，变成可被 Agent 直接调用的结构化文件。不是平台、不是 SaaS、不需要 SDK。一个目录就是知识库的全部。

单独用 GTD：三年后你有一堆已完成任务的历史记录，但没有可迁移的知识结构。
单独用 OKF：你有一个漂亮的知识库，但如果缺少"收集→理清→执行"管道，它就是知识的坟场。

它们的关系是：

```
GTD 收集 → 理清 → 执行
              ↓
          有价值的知识 → OKF 沉淀
              ↓
          下次执行时作为上下文加载
```

**GTD 是流水线，OKF 是仓库。** 流水线不停转，仓库持续积累。

## AI Agent 是这套系统的操作员

手动维护 GTD 有一个致命痛点：**级联更新。** 一件事状态变了，要同步改六份文件——收件箱、项目文件、下一步行动、等待清单、日志、周报。漏一个就产生不一致。人工做到第三天你就放弃了。

AI Agent 在这里做三件事：

**1. 无摩擦收集。** "记一下，XXX 有个坑，回头整理。"——一句话写入收件箱。不打断心流。

**2. 主动理清。** 你说"推进博客项目"，Agent 追问："下一步具体做什么？选框架还是写第一篇文章？"模糊任务被拆成可执行的物理动作。

**3. 级联更新自动化。** 你说"博客框架选了 Astro"，Agent 同时更新：项目文件状态、下一步行动、当日日志、本周回顾。六份文件，一次对话搞定。

我在 6 月 22 日的真实数据：

| 指标 | 数值 |
|------|------|
| 收件箱处理 | 6/6 全部理清 |
| 活跃项目 | 5 个 |
| 当日完成 | 7 项 |
| 级联更新触发 | 4 次，零遗漏 |
| 等待他人 | 2 项，状态透明 |

一天之内，4 次级联更新，没遗漏一个。

## 为什么这件事跟你的个人品牌有关

这是整篇文章最重要的一节。

你已经在用 AI 了。你的每一次 AI 对话都是潜在的内容资产——踩坑记录、方案对比、设计决策。但如果你没有沉淀机制，这些资产的生命周期就是一次对话的长度。

GTD + OKF + AI Agent 组合产生的不是"效率提升"，是**知识资产的复利增长**：

**第一步：GTD 确保高价值事项被推动。** 系统里有一个品牌过滤器——每个进入的事项都必须回答：这件事能提升我的个人品牌吗？增值的事推动执行，消耗的事果断砍掉。你的注意力投向，决定了三年后你在哪里。

**第二步：OKF 确保产出被沉淀。** 做完一件事，有价值的知识立即落盘成 OKF 笔记——可搜索、可引用、Agent 可直接加载为上下文。不是聊天记录，是可复用的资产。我在 OKF 里沉淀的技术笔记，3 天内被复用了 2 次。

**第三步：AI Agent 确保系统不腐烂。** 回顾、更新、级联同步全部自动化。你负责思考和创造，系统负责记忆和组织。

最终的链条：**OKF 笔记 → 博客文章 → 技术影响力 → 更好的机会。**

这篇博客本身就是这条复利链的产物。它来自 GTD 系统捕获的一个意图（"输出 GTD+OKF 分享"），OKF 知识库里的沉淀内容做素材，AI Agent 辅助整理结构。没有前面的系统，这篇文章不存在。

## 怎么开始（今天就能做）

不需要一次性搭完整体系。三件事：

**第一步：建目录**

```bash
mkdir -p ~/wealth/{inbox,projects,logs,reviews,reference}
mkdir -p ~/Documents/knowledge/{技术,方法论,项目,决策,参考}
```

**第二步：开始收集。** 对你的 AI 工具说："记一下，[任何你觉得该记录的事]。"先做三天，只收集不理清。让收件箱先胖起来。

**第三步：第一次理清。** 三天后，让 Agent 逐条处理收件箱——这条能行动吗？下一步物理动作是什么？分到哪类？理清完你会有第一份下一步行动清单。

系统不是设计出来的，是用出来的。从一条收件箱开始。

---

**关于 OKF：** [Open Knowledge Format](https://github.com/google/okf) 是 Google Cloud 在 2026 年 6 月发布的轻量知识规范。它的 SPEC.md 不到 100 行，5 分钟能读完。一个目录就是知识库的全部——可移植、可版本控制、AI Agent 可直接读写。

## 最后一句实话

GTD 有个悖论：系统搭得越完善，你越容易产生"我已经搞定了一切"的错觉，然后停止回顾。AI Agent 可以帮你省掉手动更新的麻烦，但不能替你建立习惯。

但好消息是——建立习惯需要 21 天，而 AI Agent 让前 20 天不再是苦差事。你只需要做一件事：**从一条收件箱开始，今天。**

## 附录：目录结构参考

**GTD 工作空间** (`~/wealth/`)：

```
~/wealth/
├── inbox/                  # 收件箱：未理清的原始记录
├── next-actions.md         # 下一步行动清单
├── waiting-for.md          # 等待他人清单
├── someday.md              # 将来也许清单
├── projects/               # 进行中的项目
│   └── <项目名>.md
├── reference/              # 参考资料归档
├── logs/                   # 工作日志
│   └── YYYY-MM-DD.md
└── reviews/                # 每周回顾
    └── YYYY-Ww.md
```

**OKF 知识库** (`~/Documents/knowledge/`)：

```
knowledge/
├── 技术-Tech/              # 技术踩坑、配置记录
├── 方法论-Methods/          # GTD、OKF、认知觉醒等方法论
├── 项目-Projects/           # 项目笔记，每个项目一个子目录
├── 决策-Decisions/          # 架构决策记录 (ADR)
├── 参考-References/         # 论文笔记、外部资料
└── 碎片-Fragments/          # 待深化的零散想法
```
